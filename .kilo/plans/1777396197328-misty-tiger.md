# Python CLI Tool: Directory Scanner & Task Manager

## Project Overview
Build a production-ready Python CLI tool that combines:
1. Fast recursive directory scanner with color-coded file classification
2. Integrated task manager for process monitoring
3. Interactive terminal UI using `rich` library

## Requirements Analysis

### Requirement 1: Fast & Comprehensive Directory Scanner
- **Root Path**: Default to home directory or C:\ (OS-dependent)
- **Scanning**: Recursive using `os.scandir()` for performance
- **Coverage**: Include hidden and system files
- **Progress**: Visual indicator during scanning
- **Display**: Tree-like structure using `rich.Tree`

**Implementation Strategy:**
- Use `os.scandir()` with generator pattern for memory efficiency
- Implement progress counter/bar with `rich.Progress`
- Build tree structure incrementally
- Support depth limiting to avoid excessive output

### Requirement 2: Color-Coded File Classification
Three categories with heuristics:

**RED (Unsafe - Safe to delete):**
- Temp files: `.tmp`, `.log`, `.bak`, `.temp`
- Cache dirs: `__pycache__`, `.cache`, `node_modules`
- Browser caches: `Cache`, `Cookies`, `History`
- Paths: `%TEMP%`, `%LOCALAPPDATA%\Temp`, Downloads
- Archive extracts, duplicate files

**BLUE (Critical - DO NOT DELETE):**
- System paths: `Windows\`, `System32\`, `Program Files\`
- Core files: `.dll`, `.exe` (system-signed)
- Active processes: Check against running processes
- Registry/config: System configuration files
- Service dependencies

**GREEN (System but removable):**
- `Windows.old`, `$Recycle.Bin`
- Prefetch cache, installer temp
- `.old`, `.bak` versions of system files
- Driver backup, old updates

**Classification Logic:**
- Path-based matching with regex patterns
- Extension-based heuristics
- System file flags (Windows attributes)
- Process locking detection via psutil

### Requirement 3: Integrated Task Manager
- **Separate Mode**: CLI menu to switch between Scanner/TaskManager
- **Display**: Table with PID, Name, CPU%, Memory(MB), Status
- **Functionality**: Select and terminate processes safely
- **Tool**: Use `psutil` library

**Implementation Strategy:**
- Main menu using `rich.prompt.Prompt`
- Task manager uses `rich.table.Table`
- Safe termination with error handling
- Auto-refresh capability

### Requirement 4: Technical & Safety Requirements
- **UI Framework**: `rich` for all terminal rendering
- **Process Management**: `psutil` for monitoring
- **Language**: English throughout (all code, comments, output)
- **Error Handling**: Graceful handling of PermissionError, OSError
- **Code Quality**: 
  - Type hints (PEP 484)
  - PEP 8 compliance
  - Docstrings for all functions
  - Modular design with separate classes
  - Comprehensive comments for classification logic
- **Safety**: 
  - NO auto-deletion
  - Interactive confirmation required
  - Prominent disclaimer on startup
  - Dry-run preview before any deletion

## Safety Considerations
1. **Deletion Prevention**: No auto-delete, require explicit confirmation
2. **Undo Capability**: Could implement recycle bin move (optional)
3. **Logging**: Optional warning log of attempted deletions
4. **Preview**: Show what would be deleted before confirming
5. **Rollback**: Handle exceptions gracefully if deletion fails

## Edge Cases
- Symbolic links and junctions (skip or handle specially)
- Very deep directory trees (implement depth limit)
- Large file counts (optimize with generators)
- Locked files (detect via psutil)
- Permission denied folders (log and continue)
- Network paths (optional: support or warn)
